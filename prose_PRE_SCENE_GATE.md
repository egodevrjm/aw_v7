## STEP 2.7 — SKILL USAGE GATE (hard-block — skills that MUST trigger for their scene types)

> **This step exists because session Claude consistently forgets to invoke the skills that exist to prevent the most common failures. Phone renderer is the most-forgotten skill. Others are nearly as bad. Every skill listed below is described in its own skill frontmatter as MUST-TRIGGER — and every time Claude writes a scene of the matching type without invoking the skill, a known failure mode is re-introduced.**

**Before writing a single line of prose for a scene, check the table below and invoke every applicable skill:**

| Scene type | MUST invoke | Why (what the skill prevents) |
|---|---|---|
| **Any phone interaction, text, DM, notification, group chat render, social media feed view** | `phone-renderer` | Without this: narrative contamination inside phone renders (FM-20), wrong formatting, no live-simulation behaviour, chat previews treated as prose |
| **Any media reaction, tabloid article, fan comment, press coverage, Reddit thread, TikTok caption** | `media-renderer` | Without this: article summaries instead of rendered articles, wrong voice/tone per outlet, no distinct register by medium |
| **Any exterior scene, or mid-scene location change that goes outdoors** | `pap-scale-calculator` | Without this: pap presence as celebrity-tier not infrastructure-tier (FM-07), pap count undersold, surprise/event framing for what is weather |
| **Any scene requiring Alex's physical description or outfit — dressing, leaving a property, new scene head, pap scene, fitting** | `outfit-builder` | Without this: generic "well-dressed man" prose, brand matrix ignored, wrong register for the scene |
| **Any scene where music is being made or heard — Alex plays piano, sings, rehearses, performs, a jam, a recording session** | `performance-renderer` | Without this: performance compression (FM-14), "played beautifully" gestures instead of sonic specifics, music as backdrop not subject |
| **Any scene where a professional/business document is received, read, or sent — sync offer, agency pitch, royalty statement, legal correspondence, contract, press strategy memo** | `business-renderer` | Without this: document summary instead of rendered document, wrong format/voice for type, information-level errors |
| **Any new scene — location/time/who-is-present change** | `alexworld-scene-compositor` | Without this: forgotten Kevin, undersold pap presence, information-wall violations, compressed scenes, arrived-at-location-as-if-new |

**The test before you start writing:**

- Is this scene a phone/chat/notification scene? → `phone-renderer` MUST have been invoked.
- Is Alex outside a fully private/secured space? → `pap-scale-calculator` MUST have been invoked.
- Is Alex playing, singing, performing, or is music being heard? → `performance-renderer` MUST have been invoked.
- Is this a new scene (new location/time/company)? → `alexworld-scene-compositor` MUST have been invoked.
- Is Alex getting dressed or does a new character first see him? → `outfit-builder` MUST have been invoked.
- Is a media outlet/fan account/press article being rendered? → `media-renderer` MUST have been invoked.
- Is a business/industry document being shown? → `business-renderer` MUST have been invoked.

**If you reach the first line of prose and any applicable skill has NOT been invoked: stop. Invoke the skill. Then write.**

**The phone-renderer reminder specifically:** Phone renders are **pure simulation**. No narrative beats, no character reactions, no closing prose inside the render. Scene prose stops at "he turned on his phone." Simulation takes over. The render ends — the response ends. See FM-20.
